## Decisioni tecniche

- **Formati chiusi e versionati**: ogni nuovo artefatto usa campi, vocabolari e versioni espliciti; i campi sconosciuti vengono rifiutati.
- **Determinismo verificabile**: JSON canonico, inventari ordinati, tar USTAR normalizzati, checksum completi e fixture riproducibili.
- **No-overwrite / no-replace**: destinazioni esistenti, collisioni concorrenti e symlink non vengono sovrascritti.
- **Verifica indipendente**: gli archivi vengono validati dopo la creazione e nuovamente da un consumer separato o da un verificatore in destinazione pulita.
- **Local-first e loopback-only**: nessun servizio remoto; i launcher servono esclusivamente su `127.0.0.1`.
- **Nessun privilegio o modifica macchina**: niente installer, elevazione, servizio, registry, firewall, scheduled task o policy changes.
- **Separazione dei dati**: evaluator-session, incident log e sintesi non importano learner evidence e non introducono identità del discente.
- **Privacy by construction**: nessun nome, classe, scuola, risposta, account, timestamp, IP, path assoluto, dettaglio dispositivo o testo libero nei nuovi record operativi.
- **Stop fail-closed**: severità critica, violazione del confine privacy o runtime non sicuro richiedono stop e verifica del cleanup.
- **Parità multipiattaforma esplicita**: Linux e Windows sono eseguiti in CI; macOS usa il percorso POSIX documentato e resta una verifica manuale esplicita.

## Problemi incontrati e soluzioni

### Descriptor temporaneo su Windows

Nel synthesis kit, il file descriptor restituito da `tempfile.mkstemp` poteva restare aperto prima della riapertura del path su Windows.

**Soluzione:** chiusura esplicita del descriptor prima della scrittura successiva e aggiunta di un test regressivo dedicato. La correzione è inclusa nel merge `4fdde23`; Linux run 824 e Windows run 68 sono verdi.

### Rischi di path, symlink e collisioni

L'espansione dei workflow locali aumentava il rischio di traversal, alias di path, symlink, file speciali e sostituzione accidentale degli output.

**Soluzione:** path POSIX canonici, contenimento nel root, rifiuto di symlink e membri speciali, verifica di file regolari, installazione no-replace, cleanup di proprietà e test di collisione/tampering.

### Ambiguità tra dati del discente e dati operativi

Record di sessione, incidenti e sintesi avrebbero potuto trasformarsi in telemetria o contenere informazioni identificanti.

**Soluzione:** contratti chiusi privi di campi liberi, identificativi e temporali; separazione esplicita da learner evidence; vocabolari approvati; nessun upload o scansione automatica.

### Run iniziali `skipped`

Alcuni commit contrattuali o stati preliminari hanno prodotto run `skipped` per effetto dei trigger e delle regole di consumo CI.

**Soluzione:** nessuna correzione di prodotto necessaria. Ogni incremento mergiato è stato validato successivamente da run Linux/Windows verdi. Per la PR #95 le run autorevoli finali sono 842 e 86, entrambe verdi.
