# Why Raiatea Is Not Just an LLM Interface

**Status:** Draft  
**Version:** 0.1.0  
**Project phase:** Project Genesis — Leaving the Shore  
**Purpose:** Define the objective value Raiatea must provide beyond what a capable general-purpose LLM can already do.

---

## 1. The hard question

A user who already subscribes to ChatGPT or another advanced LLM can ask it to:

- explain a topic;
- search the web;
- analyse files;
- translate a document;
- create a study plan;
- produce a report;
- compare sources;
- generate exercises;
- summarise news;
- recommend books;
- help write an article or paper.

If Raiatea only adds a themed interface around these same capabilities, it has no durable reason to exist.

> Raiatea is justified only if it turns isolated model outputs into durable, verifiable, evolving, portable, and shared knowledge work.

---

## 2. The honest starting point

A strong LLM should remain one of Raiatea's primary engines.

Raiatea should not compete with frontier model providers on raw language generation, general reasoning, multimodal understanding, or model training.

It should be model-agnostic and allow users to employ:

- commercial frontier models;
- local models;
- specialised scientific models;
- different providers for verification;
- future models not yet available.

The question is therefore not:

> Is Raiatea smarter than ChatGPT?

It is:

> What persistent system should surround an LLM so that its intelligence becomes trustworthy, cumulative, collaborative, and useful over years?

---

## 3. Chat interaction versus knowledge infrastructure

A general LLM primarily produces an answer for the current interaction.

Raiatea must produce and maintain a structured knowledge object.

```text
Standalone LLM
question → generated answer

Raiatea
question
→ expedition
→ corpus and source inventory
→ claims, evidence, events, systems, and processes
→ bibliography and provenance
→ durable artefacts
→ progress and user model
→ public contribution
→ continuous monitoring
→ revisions and historical versions
```

A response may disappear into conversation history. A Raiatea artefact must remain inspectable, reusable, exportable, updateable, and connected to all later work.

---

## 4. Objective differentiators

### 4.1 Durable domain model

Raiatea stores more than prose.

It maintains typed and versioned objects such as:

- entities;
- claims;
- observations;
- evidence;
- events;
- states;
- processes;
- causal hypotheses;
- predictions;
- contradictions;
- source roles;
- uncertainty;
- revisions.

The same knowledge can therefore be viewed as a profile, timeline, map, dossier, study route, bibliography, or forecast without asking a model to reconstruct everything from scratch.

### 4.2 Persistent provenance

Every important conclusion remains linked to exact source coordinates, acquisition history, transformations, model versions, reviews, and corrections.

The user can move from a statement to:

```text
statement
→ supporting and opposing evidence
→ source fragment
→ original document
→ extraction method
→ later corrections
```

Citations are part of the data model, not decorative text appended to a generated answer.

### 4.3 Longitudinal evolution

Raiatea preserves how knowledge changed.

It can answer:

- What did we believe six months ago?
- Which new evidence changed the model?
- Which source was corrected or retracted?
- When did a public actor change position?
- Which prediction succeeded or failed?
- Which learning route became obsolete?

A current answer alone cannot reveal the trajectory unless the entire history has been deliberately preserved and modelled.

### 4.4 Continuous watch tied to prior understanding

A general news summary tells the user what happened.

Raiatea should tell the user:

- what changed relative to their specific route;
- which previous explanation is affected;
- whether the change confirms, contradicts, extends, or supersedes it;
- what should be read or revised next.

This comparison against a durable prior model is one of the strongest recurring values.

### 4.5 Private corpus and public commons

Raiatea separates:

- private books, documents, translations, notes, and interests;
- shareable derived knowledge;
- public source references;
- collectively verified knowledge bundles.

One user's private study can improve future public expeditions without publishing the private library.

A standalone LLM session does not normally create a governed, federated knowledge commons from every lawful private expedition.

### 4.6 Community verification and structured disagreement

Raiatea should support:

- competing claims;
- evidence for and against;
- source criticism;
- minority reports;
- replication;
- expert review;
- public corrections;
- principled forks.

A single generated response usually collapses uncertainty into one narrative. Raiatea preserves disagreement as first-class knowledge.

### 4.7 Reproducible expeditions

A user can publish the route they followed:

- ordered sources;
- prerequisites;
- decisions;
- exercises;
- experiments;
- models constructed;
- open questions.

Another person can clone the route, acquire the cited private sources, use public alternatives, and improve it.

The product is not only an answer; it is a reproducible method for arriving at understanding.

### 4.8 Personal learning and research state

Raiatea should maintain an explicit model of:

- what the user already knows;
- which evidence they studied;
- misunderstandings and gaps;
- completed exercises;
- unresolved questions;
- preferred depth and available time;
- previous decisions and their outcomes.

This permits precise adaptation without relying on a vague conversational memory.

### 4.9 Provider independence and portability

Raiatea's corpus, schemas, routes, and public knowledge should not depend on one model provider.

The same expedition can be processed by several models, compared, verified, exported, and continued elsewhere.

The user keeps the map even if a subscription, provider, model, or company changes.

### 4.10 Public-interest guarantees

Raiatea embeds constitutional rules about:

- free civic access;
- emergency open access;
- no paid factual ranking;
- transparency logs;
- source and commercial disclosures;
- user ownership and export;
- preservation of dissent.

A commercial general-purpose model may behave responsibly, but its product terms are not the same thing as a federated public-interest constitution.

---

## 5. What Raiatea must not claim

Raiatea should not claim that an LLM cannot:

- produce excellent explanations;
- run deep web research;
- analyse uploaded files;
- remember useful context;
- connect external applications;
- create polished reports;
- perform powerful personalised work.

These capabilities will continue improving.

Raiatea's advantage cannot depend on a temporary limitation of current models.

It must depend on architecture and governance that remain valuable even when LLMs become far more capable.

---

## 6. The product relationship

The ideal relationship is:

```text
LLM = navigator, analyst, translator, writer, and reasoning engine

Raiatea = vessel, chart room, logbook, observatory, archive,
           verification network, public commons, and expedition protocol
```

ChatGPT or another model may do much of the intellectual work inside Raiatea.

Raiatea coordinates that work across time, sources, people, models, rights, and communities.

---

## 7. A concrete comparison

### Task: “Help me become an AI engineer and understand LLMs deeply.”

A capable LLM can immediately generate:

- a roadmap;
- book recommendations;
- explanations;
- exercises;
- answers to questions;
- web research reports.

Raiatea must additionally provide:

1. An inventory of the user's actual books, papers, repositories, videos, and prior knowledge.
2. A source-backed domain map and historical evolution.
3. Original and Italian translation views for privately owned works.
4. A route that adapts after each exercise and document studied.
5. Public profiles of influential people and their evolving positions.
6. Reproducible experiments and evidence linked to code versions.
7. A persistent bibliography with source roles and acquisition guidance.
8. A daily or weekly delta explaining what changed in the field and in the route.
9. Community corrections, alternative models, and verified contributions.
10. Exportable artefacts and continuity across model providers.

If Raiatea cannot provide these reliably, the user should use the LLM directly.

---

## 8. Immediate value versus cumulative value

### Immediate value

Raiatea must save substantial work in the first session by producing a durable deliverable:

- a living bilingual chapter;
- a research brief;
- an evidence dossier;
- a source inventory and gap analysis;
- a personalised field guide;
- an update briefing tied to an existing route.

### Cumulative value

Over time Raiatea accumulates:

- verified routes;
- longitudinal history;
- personal learning state;
- reusable translations and explanations;
- public knowledge bundles;
- resolved forecasts;
- community trust;
- deduplicated processing;
- continuously improving maps.

The cumulative value must arise from preserved structure, not from user lock-in.

---

## 9. Falsifiable product tests

Raiatea should be compared directly against using a frontier LLM alone.

For each target workflow, run controlled tests measuring:

- time to a usable deliverable;
- factual and citation errors;
- source coverage;
- ability to reproduce the result;
- quality after one month of updates;
- ability to identify exactly what changed;
- learning retention and transfer;
- ease of correcting one claim everywhere it appears;
- portability to another model provider;
- value of community contributions;
- attention saved by continuous briefing.

A proposed feature is justified only when Raiatea measurably outperforms a well-prompted LLM workflow on one or more important dimensions.

---

## 10. Kill criterion

Raiatea should adopt a strict rule:

> If a user can obtain an equally accurate, durable, reproducible, updateable, portable, and well-governed result from a general LLM with substantially less effort or cost, Raiatea should not build that feature.

This prevents Raiatea from becoming a slower imitation of ChatGPT.

---

## 11. Strategic focus

The first product should not attempt to beat ChatGPT at conversation.

It should solve a workflow where conversation alone becomes burdensome:

> Turn a real private corpus and a long-term learning goal into a living, bilingual, source-linked expedition that remains updated and contributes lawful derived knowledge to a shared network.

This is narrow enough to test and broad enough to exercise Raiatea's unique architecture.

---

## 12. Provisional value proposition

> Use an LLM when you need an excellent answer now. Use Raiatea when the answer must become a durable map that can be verified, revised, followed, shared, and kept alive over time.

Or more precisely:

> Raiatea turns the intelligence of interchangeable LLMs into persistent, evidence-linked, evolving knowledge that belongs to its users and improves through the work of a community.
